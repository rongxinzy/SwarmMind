# Product Positioning

## One-line Positioning

SwarmMind is an open-source **enterprise agent control plane** that turns fragmented internal systems into governed context for reliable agent execution.

## Who It Is For

- Mid-to-large organizations running complex knowledge workflows
- Teams with fragmented internal data and strict compliance requirements
- Leaders who need direct, explainable answers from enterprise systems

## Core Problems

1. **Data fragmentation**: business context spread across siloed systems
2. **Execution risk**: uncontrolled agent actions can violate policy
3. **Trust gap**: outputs without provenance and auditability are not adoptable
4. **Operational drag**: humans manually stitch context before decisions

## Product Boundaries

SwarmMind focuses on:

- context governance,
- policy-driven routing,
- auditable execution,
- human supervision and approval,
- enterprise connector integration.

SwarmMind does **not** focus on:

- entertainment chatbots,
- pure prompt playgrounds,
- "many agents talking" demos without governance.

## Differentiation

### From Multi-Agent Frameworks

Most frameworks optimize collaboration mechanics; SwarmMind optimizes enterprise control, trust, and execution reliability.

### From Workflow Engines

Traditional workflow tools enforce static state machines; SwarmMind manages evolving, semantically rich context with governance metadata.

## Messaging Pillars

1. **Governed Context Layer** over shared memory
2. **Execution Trace** over opaque reasoning claims
3. **Policy Learning** over brittle manual routing rules
4. **Context-to-View Compiler** over unstructured status generation

## North-star Outcome

Move organizations from "human-operated workflows with AI assistance" to "agent-operated workflows with human supervision."
