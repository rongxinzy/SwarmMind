# 项目体系与项目空间

> 日期：2026-03-27
> 用途：定义从项目创建到项目执行、看板、产物和设置的整套页面

## 0. 项目空间固定结构

### 0.1 设计结论

`Project` 不是若干离散 tab 的集合，而是一个固定信息壳层下的正式工作空间。

所有项目页必须共享同一骨架，避免后续出现：

- 每新增一种数据就加一个新 tab
- 同一个项目在不同角色下长成完全不同的页面
- 用户进入项目后不知道“现在应该先看哪里”

### 0.2 项目空间固定骨架

```text
+------------------------------------------------------------------------------------------------+
| 项目顶栏：项目名 / 状态 / 当前阶段 / 主动作                                                    |
| 项目状态条：阻塞 / 待审批 / 风险 / 活跃 Team                                                   |
| 二级导航：[总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                  |
+------------------------------------------------------------------------------------------------+
| 当前子页内容                                                                                    |
+------------------------------------------------------------------------------------------------+
```

固定规则：

- 二级导航顺序固定。
- 权限影响内容深度，不改变导航结构。
- 新能力优先归入现有六个子页，不轻易新增平行页面。
- 跨项目治理动作，例如统一审批，优先放在项目外的治理入口中。

### 0.3 六个子页各自回答的问题

| 子页 | 用户核心问题 | 默认对象 |
|------|--------------|----------|
| 总览 | 这个项目现在整体什么状态？ | 所有人 |
| 协作台 | 我要如何继续推动它？ | 项目成员 |
| 看板 | 任务具体流到哪一步了？ | 项目成员 / 负责人 |
| 时间线与风险 | 节奏、依赖和风险在哪里？ | 负责人 / 管理者 |
| 产物 | 已经沉淀了什么正式输出？ | 有访问权限的成员 |
| 设置 | 这个项目允许谁参与、用什么能力？ | 负责人 / 管理员 |

### 0.4 项目顶栏草图

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设      进行中 / 需求澄清      [进入协作台] [上传资料] [更多操作]             |
| 目标：构建企业统一 CRM，首期交付销售 CRM MVP                                                  |
| 阻塞：1   待审批：1   风险：medium   活跃 Team：软件开发 Team                                  |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------------------------------------------------------------------------+
```

### 0.5 导航规则

#### 规则 1：总览永远是默认落点

- 新建项目成功后，默认进入 `总览`。
- 从工作台或项目列表进入项目，也默认先落 `总览`。

#### 规则 2：主动作优先导向协作台

- 顶栏右上角主动作优先是 `进入协作台`。
- 因为项目空间的核心不是被动浏览，而是推动工作继续。

#### 规则 3：状态条始终可见

- 阻塞、待审批和风险不应藏到子页内部。
- 用户在任何项目子页都应该知道当前项目是否卡住。

#### 规则 4：右侧治理位按子页复用

- 总览：显示摘要和最近决策
- 协作台：显示 InterfaceDecision、thread、上下文
- 看板：显示任务详情
- 时间线与风险：显示风险详情
- 产物：显示 artifact 元数据
- 设置：显示影响说明

## 1. 页面一：项目列表

### 1.1 页面卡片

- 页面名：`项目列表`
- 目标用户：所有有项目访问权限的成员
- 页面目标：浏览、筛选、进入正式项目空间
- 页面入口：侧边栏 `项目`
- 核心动作：
  - 查看项目状态
  - 新建项目
  - 按 Team、状态、风险筛选

### 1.2 信息结构

- 顶部：搜索与筛选
- 中部：项目表格 / 卡片
- 右侧：项目摘要详情

### 1.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 项目                                                           [搜索项目] [筛选] [新建项目]    |
+------------------------------------------------------------------------------------------------+
| 名称                    | Team              | 当前阶段     | 状态       | 风险     | 操作     |
| 新一代 CRM 平台建设     | 软件开发 Team     | 需求澄清     | 进行中     | medium   | 打开     |
| 招聘流程优化            | 招聘 Team         | 面试安排     | 等待审批   | high     | 打开     |
| 销售培训体系            | 销售培训 Team     | 验收         | 已完成     | low      | 打开     |
+------------------------------------------------------------------------------------------------+
| 右侧摘要：项目目标 / 负责人 / 最近 artifact / 待审批数量 / 当前阻塞                            |
+------------------------------------------------------------------------------------------------+
```

### 1.4 关键交互

1. 点击项目行进入项目总览。
2. 点击 `新建项目` 进入创建页。
3. 筛选项支持 `状态 / Team / 风险 / 最近更新`。

### 1.5 状态设计

- 空状态：引导创建第一个项目
- 加载状态：表格骨架
- 成功状态：支持批量浏览和快速筛选
- 错误状态：项目列表与右侧摘要分别可重试
- 权限不足状态：隐藏无权限项目，只保留可访问项目

### 1.6 移动端规则

- 表格改成项目卡片流
- 点击卡片进入项目总览

### 2.4 关键交互

1. 从 ChatSession 提升进入时，目标、范围、约束自动预填。
2. 选择 Team 后，右侧同步显示默认 workflow 说明。
3. 点击 `创建项目` 后直接进入项目总览。

### 2.5 状态设计

- 空状态：展示内置 Team 的典型用途
- 加载状态：Team 列表和默认 workflow 预览加载中
- 成功状态：创建后展示“项目已建立，Team 入口已上线”
- 错误状态：字段校验错误、Team 加载错误、创建失败
- 权限不足状态：没有项目创建权限时只展示申请入口

### 2.6 移动端规则

- 单列表单
- Team 预览折叠在下方卡片中
- `创建项目` 固定底部

## 3. 页面三：项目总览

### 3.1 页面卡片

- 页面名：`项目总览`
- 目标用户：项目成员、审批人、管理者
- 页面目标：用一个页面概览项目状态、阶段、阻塞、产物和成员
- 页面入口：项目列表、工作台
- 核心动作：
  - 查看当前阶段
  - 打开项目协作台
  - 进入看板、时间线、产物
  - 发现阻塞与审批

### 3.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航
- 中部：阶段摘要卡、阻塞卡、待审批卡、成员卡
- 下部：最近任务、最近 artifact、运行时间线摘要

### 3.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
```

### 3.4 关键交互

1. 点击 `进入协作台`，进入 `TeamInterfaceAgent` 入口页。
2. 点击阻塞卡或待审批卡，跳到审批详情。
3. 点击最近 artifact，进入项目产物库对应条目。

### 3.5 状态设计

- 空状态：新项目尚未启动，提示“向接口人 Agent 说明需求”
- 加载状态：各摘要卡独立骨架
- 成功状态：展示阶段、任务、artifact、成员健康度
- 错误状态：局部模块失败不阻断整个总览
- 权限不足状态：隐藏受限 artifact 和成员信息

### 3.6 移动端规则

- 摘要卡单列排列
- 最近任务与最近产物拆成折叠区

## 4. 页面四：项目协作台

### 4.1 页面卡片

- 页面名：`项目协作台`
- 目标用户：项目成员
- 页面目标：作为 `TeamInterfaceAgent` 的统一入口，承接需求、干预、补充和派发
- 页面入口：项目总览 `进入协作台`
- 核心动作：
  - 向接口人 Agent 提交指令
  - 查看工作流归属与 thread 决策
  - 追踪当前活跃 Agent
  - 触发新任务或调整范围

### 4.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航
- 左侧：Workstream 列表
- 中间：项目对话 / 活动流
- 右侧：上下文摘要、当前 thread、路由决策

### 4.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 新一代 CRM 平台建设 / 协作台                                  [上传资料] [查看运行日志]       |
| 阻塞：1   待审批：1   风险：medium   活跃 Team：软件开发 Team                                  |
| [总览] [协作台] [看板] [时间线与风险] [产物] [设置]                                            |
+------------------------------+--------------------------------+--------------------------------+
| Workstreams                  | 协作流                         | 上下文与治理                     |
| - 项目启动                    | 用户：MVP 先别做复杂报表      | 当前 Workstream：项目启动        |
| - 权限与知识库接入            | InterfaceAgent：已调整范围    | target_agent：产品经理 Agent     |
| - 前端信息架构                | 并生成新任务“基础销售漏斗汇总”| decision：reuse_thread           |
|                                |                                | thread：ws-init-003              |
|                                |                                | 可见级别：project-internal       |
|                                | [输入框......................................................] |
|                                |                                | [新建工作流] [查看线程详情]      |
+------------------------------+--------------------------------+--------------------------------+
```

### 4.4 关键交互

1. 用户在输入框提交指令，右栏立即显示 `InterfaceDecision` 摘要。
2. 如果命中新工作流，左侧自动新增 Workstream。
3. 若触发高风险能力，顶部插入“等待审批”条。

### 4.5 状态设计

- 空状态：只显示项目启动提示和示例输入
- 加载状态：活动流流式刷新，右栏显示路由中
- 成功状态：可以看到指令如何变成任务与工作流
- 错误状态：路由失败、thread 冲突、权限不足分别提示
- 权限不足状态：没有 `project.run` 时输入框禁用

### 4.6 移动端规则

- 左侧 Workstream 改成顶部 tabs
- 右栏改为底部抽屉

- 页面目标：以最小可视化方式承接 workflow 执行
- 页面入口：项目内导航

  - 查看任务流转

  - 发现阻塞
  - 打开任务详情与相关 artifact

### 5.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航、视图切换与筛选
- 中部：Kanban 列
- 右侧：任务详情抽屉

### 5.3 桌面端线框

### 5.4 关键交互

1. 点击卡片打开任务详情，查看负责人、来源 Workstream、关联 artifact。

3. 看板变化同步回写项目总览。

### 5.5 状态设计

- 空状态：提示从协作台输入目标，系统将自动生成首批任务
- 加载状态：列骨架 + 卡片占位
- 成功状态：按 workflow 投影任务
- 错误状态：保存排序失败、详情加载失败
- 权限不足状态：只读成员无法拖拽

### 5.6 移动端规则

- 看板改成按列纵向浏览
- 任务详情独立二级页

- 空状态：提示“项目推进后，产物会自动归档到这里”
- 加载状态：列表与预览分别骨架
- 成功状态：可追溯到任务、run、审批
- 错误状态：预览失败时保留下载入口
- 权限不足状态：只显示元数据，不开放内容预览

### 7.6 移动端规则

- 预览改为全屏查看器
- 元数据折叠到底部抽屉

## 8. 页面八：项目设置 / Team 挂载

### 8.1 页面卡片

- 页面名：`项目设置`
- 目标用户：项目负责人、管理员
- 页面目标：管理成员、可见范围、Team 挂载、项目级 MCP allowlist
- 页面入口：项目页 `更多操作`
- 核心动作：
  - 编辑项目约束

### 8.2 信息结构

- 顶部：项目顶栏、状态条、固定二级导航

- 中间：当前设置表单
- 右侧：影响说明
