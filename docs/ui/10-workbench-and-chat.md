# 工作台与轻量会话

> 日期：2026-03-27
> 用途：定义首页、轻量聊天入口和最近记录页

## 1. 页面一：工作台

### 1.1 页面卡片

- 页面名：`工作台`
- 目标用户：所有登录用户
- 页面目标：作为每个用户的权限化驾驶舱，提供首页总览、快速发起和待办聚合
- 页面入口：侧边栏 `工作台`
- 核心动作：
  - 新建对话
  - 跳转最近项目
  - 查看待审批项
  - 查看运行中的项目

### 1.2 信息结构

- 左侧：全局导航
- 中上：欢迎区与主输入框
- 中下左：最近项目 / 最近会话
- 中下中：待审批 / 风险提示
- 中下右：运行中 Team / 今日计划
- 顶部可扩展：管理视角下出现组织级统计卡

### 1.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 工作台                                        [我的范围 v] [全局搜索] [新建项目]              |
+------------------------------+-----------------------------------------------------------------+
| 侧边栏                       | 你好，Kr Li                                                      |
|                              | 今天你要推进什么？                                               |
|                              | +-------------------------------------------------------------+  |
|                              | | 输入一句目标，先开启临时对话或跳转到已有项目               |  |
|                              | +-------------------------------------------------------------+  |
|                              | [新建对话] [新建项目] [查看待审批]                              |
|                              |-----------------------------------------------------------------|
|                              | 最近项目               | 待审批                | 正在运行的 Team |
|                              | - 新一代 CRM 平台建设  | - KB 访问审批         | - 软件开发 Team |
|                              | - 招聘流程优化         | - GitLab 写权限审批   | - 招聘 Team     |
|                              |-----------------------------------------------------------------|
|                              | 最近会话               | 今日摘要                                 |
|                              | - 昨天的需求讨论       | 3 个项目推进中，1 个阻塞，2 个审批待处理 |
|                              | - GitLab 接入方案      |                                           |
+------------------------------+-----------------------------------------------------------------+
```

### 1.3.1 管理视角扩展示意

同一页面在高权限下增加组织级卡片，而不是跳到另一套页面：

```text
+------------------------------------------------------------------------------------------------+
| 工作台                                        [全组织 v] [全局搜索] [新建项目]                |
+------------------------------+-----------------------------------------------------------------+
| 侧边栏                       | 组织级摘要：12 个项目进行中 / 3 个阻塞 / 5 个待审批             |
|                              |-----------------------------------------------------------------|
|                              | 项目健康表             | 风险热区              | Team 负载          |
|                              | CRM 平台建设  medium   | 招聘流程优化  high   | 软件开发 Team  6   |
|                              | 销售培训体系  low      | SSO 依赖       medium | 招聘 Team      4   |
+------------------------------+-----------------------------------------------------------------+
```

### 1.4 关键交互

1. 在主输入框输入目标，系统先判断是否进入 `新建对话`。
2. 点击 `新建项目`，进入项目创建页。
3. 点击待审批卡片，跳转 `审批中心` 并自动过滤到“与我相关”。
4. 点击最近项目，直接进入项目总览。
5. 高权限用户可切换 `我的范围 / 部门范围 / 全组织` 数据视角。

### 1.5 状态设计

- 空状态：展示引导卡，说明 `新建对话` 与 `新建项目` 的差别
- 加载状态：最近列表和摘要卡使用骨架屏
- 成功状态：显示今日项目健康度摘要
- 错误状态：某个聚合模块加载失败时仅局部报错
- 权限不足状态：没有审批权限时隐藏审批汇总，只保留只读统计

### 1.6 权限分层规则

- 普通成员：
  - 看到与自己相关的项目、最近会话、待我处理审批
- 项目负责人：
  - 额外看到所负责项目的健康摘要、阻塞和风险
- 管理者 / 高层：
  - 额外看到跨项目统计、风险热区、Team 负载和审批堆积

因此，`工作台` 是统一页面，不是多个角色各做一套页面。

### 1.7 卡片体系

工作台卡片建议拆成五类，避免后面越做越像杂乱信息墙。

#### A. 发起类卡片

- 快速输入框
- 新建对话
- 新建项目
- 常用模板入口

职责：

- 承接用户意图输入
- 让首页始终可发起，而不只是看报表

#### B. 执行类卡片

- 最近项目
- 正在运行的 Team
- 我的活跃 Workstream
- 今日关键进展

职责：

- 告诉用户“哪些事情正在推进”
- 让用户一跳回到执行上下文

#### C. 治理类卡片

- 待我审批
- 审批堆积
- 阻塞项目
- 风险热区

职责：

- 告诉用户“哪些事情需要决策或解除阻塞”
- 高于普通动态流，优先级必须更高

#### D. 资产类卡片

- 最近 artifact
- 最近引用知识
- 定时任务结果

职责：

- 承接用户最近产生和消费的正式资产
- 让资产不被埋在项目内部

#### E. 组织类卡片

- 项目健康表
- Team 负载
- 部门项目分布
- 本周完成里程碑

职责：

- 只在更高权限下出现
- 提供横向比较，而不是单项目详情

### 1.8 默认卡片组合

#### 普通成员默认布局

```text
+-----------------------------------------------------------------------------------------------+
| 快速发起                                                                                      |
| 最近项目                         | 最近会话                                                   |
| 我的待处理                       | 我的活跃 Workstream                                        |
| 最近 artifact                    | 今日关键进展                                               |
+-----------------------------------------------------------------------------------------------+
```

默认原则：

- 强调回到任务现场
- 少展示组织级聚合
- 避免让普通成员被无关统计干扰

#### 项目负责人默认布局

```text
+-----------------------------------------------------------------------------------------------+
| 快速发起                                                                                      |
| 我负责的项目健康度               | 阻塞与风险                                                 |
| 待我审批                         | 正在运行的 Team                                            |
| 最近 artifact                    | 今日关键进展                                               |
+-----------------------------------------------------------------------------------------------+
```

默认原则：

- 强调项目推进、阻塞和治理
- 项目负责人的首页应先看到异常，再看到历史

#### 管理者默认布局

```text
+-----------------------------------------------------------------------------------------------+
| 组织级摘要                                                                                     |
| 项目健康表                       | 风险热区                                                   |
| 审批堆积                         | Team 负载                                                  |
| 重点项目                         | 本周完成里程碑                                             |
+-----------------------------------------------------------------------------------------------+
```

默认原则：

- 强调横向扫描与异常发现
- 不默认把大量单人会话信息放在首屏

### 1.9 卡片显示规则

#### 规则 1：固定优先级

优先级从高到低建议为：

1. 治理类卡片
2. 执行类卡片
3. 发起类卡片
4. 资产类卡片
5. 组织类卡片

说明：

- 这里的“组织类”优先级低，不代表不重要。
- 只是它们更适合在高权限视角下作为第二屏或右侧区域，而不是压掉待审批和阻塞信息。

#### 规则 2：空数据降级

- 某张卡片无数据时，优先隐藏。
- 如果卡片对应的是关键功能入口，例如 `新建项目`，则保留入口态，不保留空列表态。

#### 规则 3：权限先于个性化

- 用户可以调整卡片顺序和折叠状态。
- 用户不能通过个性化设置看到超出权限范围的数据。

#### 规则 4：范围切换必须稳定

- 范围切换只改变数据集，不改变页面结构。
- 不允许从 `我的范围` 切到 `全组织` 后变成另一套信息架构。

### 1.10 推荐卡片明细

| 卡片名 | 类型 | 最低显示角色 | 点击后去向 |
|--------|------|--------------|------------|
| 快速发起 | 发起类 | 全员 | 新建对话 / 新建项目 |
| 最近项目 | 执行类 | 全员 | 项目总览 |
| 最近会话 | 执行类 | 全员 | ChatSession |
| 我的活跃 Workstream | 执行类 | 全员 | 项目协作台 |
| 待我审批 | 治理类 | 有审批任务的成员 | 审批中心 |
| 阻塞与风险 | 治理类 | 项目负责人起 | 项目时间线与风险 / 项目总览 |
| 正在运行的 Team | 执行类 | 项目负责人起 | 项目协作台 / 项目总览 |
| 最近 artifact | 资产类 | 全员 | 资源库 / 项目产物库 |
| 项目健康表 | 组织类 | 管理者起 | 项目列表 / 项目总览 |
| 风险热区 | 治理类 | 管理者起 | 项目时间线与风险 / 审批中心 |

### 1.11 自定义规则

允许用户做的自定义：

- 折叠非核心卡片
- 调整同优先级卡片顺序
- 固定一个常用项目到首页

不允许用户做的自定义：

- 删除系统强制卡片，例如快速发起
- 将审批和阻塞类卡片完全移除出默认首屏
- 保存越权的数据范围

### 1.12 移动端规则

- 首屏保留 `快速发起`、`待处理事项`、`最近项目`
- 组织类卡片默认收进“查看更多”
- 卡片顺序仍遵循同一优先级，不做角色专属独立页面
- 全局搜索放入顶部折叠菜单

## 2. 页面二：新建对话 / ChatSession

### 2.1 页面卡片

- 页面名：`新建对话`
- 目标用户：所有登录用户
- 页面目标：承接探索性、个人化、低治理成本的初始需求输入
- 页面入口：侧边栏 `+ 新建对话`、工作台主输入框
- 核心动作：
  - 发起探索性需求
  - 上传参考资料
  - 生成摘要
  - 提升为 Project

### 2.2 信息结构

- 左侧：最近会话列表
- 中间：对话区
- 右侧：摘要与提升卡

### 2.3 桌面端线框

```text
+------------------------------------------------------------------------------------------------+
| 新建对话                                                     [上传资料] [提升为 Project]      |
+------------------------------+--------------------------------+--------------------------------+
| 最近会话                     | 对话区                         | 摘要区                         |
| - CRM 需求探索               | 用户：我们想做一个 CRM MVP... | 目标：销售 CRM MVP             |
| - 招聘流程设计               | Agent：我先整理目标和约束...  | 范围：客户/联系人/商机...      |
|                              |                                | 约束：SSO / 3 个月 / Web 优先  |
|                              | [输入框......................................................] |
|                              |                                | [生成摘要] [提升为 Project]    |
+------------------------------+--------------------------------+--------------------------------+
```

### 2.4 关键交互

1. 用户输入目标，系统实时生成结构化摘要。
2. 用户上传材料后，右侧摘要卡追加“已引用资料”。
3. 点击 `提升为 Project`，进入新建项目页并预填目标、范围、约束。

### 2.5 状态设计

- 空状态：展示示例 prompt 和“何时该提升为项目”
- 加载状态：对话流式输出，摘要区显示提炼中
- 成功状态：摘要卡可直接复用
- 错误状态：对话失败可重试；摘要提取失败不影响继续聊天
- 权限不足状态：无项目创建权限时隐藏 `提升为 Project`

### 2.6 移动端规则

- 左侧会话列表收为抽屉
- 摘要区放到底部抽屉
- `提升为 Project` 固定在底部操作栏

## 3. 页面三：最近记录

### 3.1 页面卡片

- 页面名：`最近记录`
- 目标用户：所有登录用户
- 页面目标：集中浏览最近访问的对话、项目和 artifact
- 页面入口：侧边栏 `最近`
- 核心动作：
  - 回到最近会话
  - 回到最近项目
  - 快速清理无用记录

### 3.2 信息结构

- 顶部：筛选器
- 中部：按类型分组的最近记录列表
- 右侧：所选记录详情

### 3.3 桌面端线框

```text
+-----------------------------------------------------------------------------------------------+
| 最近记录                                              [全部] [对话] [项目] [Artifact]        |
+--------------------------------+----------------------------------------------------------------+
| 列表                           | 详情                                                           |
| 今天                           | 标题：昨天的需求讨论                                          |
| - 昨天的需求讨论               | 类型：ChatSession                                              |
| - 新一代 CRM 平台建设          | 最近动作：已提升为 Project                                    |
| 昨天                           | 关联：Project / 新一代 CRM 平台建设                           |
| - GitLab 接入方案              | [打开] [从最近中移除]                                         |
+--------------------------------+----------------------------------------------------------------+
```

### 3.4 关键交互

1. 用户按类型筛选最近项。
2. 点击记录后，右侧展示来源、最近动作、关联对象。
3. 允许“从最近中移除”，但不删除真实对象。

### 3.5 状态设计

- 空状态：提示用户从工作台发起第一条对话或进入第一个项目
- 加载状态：分组列表使用骨架行
- 成功状态：自动按时间分组
- 错误状态：仅影响最近模块，不影响其他页面
- 权限不足状态：若关联对象已无权限，只显示“你已无访问权限”

### 3.6 移动端规则

- 详情区变为点击后二级页
- 筛选器改为横向滚动 chips
