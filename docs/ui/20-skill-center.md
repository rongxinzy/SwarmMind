# 技能中心

> 日期：2026-03-27
> 用途：定义 `MCP`、`Skill` 和绑定关系的统一治理入口

## 1. 页面一：技能中心

### 1.1 页面卡片

- 页面名：`技能中心`
- 目标用户：管理员、高级用户、项目负责人
- 页面目标：统一管理 `MCP`、`Skill` 和绑定关系
- 页面入口：侧边栏 `探索技能`
- 核心动作：
  - 浏览已接入 MCP
  - 浏览已安装 Skills
  - 建立绑定与解绑
  - 查看最近调用与风险级别

### 1.2 页面结构

- 左侧：全局导航
- 顶部：搜索、筛选和主动作
- 中间：二级标签 `MCP / Skills / Bindings`
- 主体：列表区 + 详情区

### 1.6 关键交互

1. 在 `MCP` 标签中选中某个 endpoint，点击 `绑定到 Team`。
2. 弹出绑定抽屉，选择目标和允许的工具命名空间。
3. 在 `Bindings` 标签中查看是否已经在某个项目里生效。
4. 高风险绑定在保存时显示审批提示。

### 1.8 状态设计

- 空状态：引导接入第一个 MCP 或安装第一个 Skill
- 加载状态：列表和详情区均使用骨架屏
- 成功状态：详情区展示最近调用统计与生效对象
- 错误状态：连接失败、元数据加载失败、绑定失败均要单独提示
- 权限不足状态：页面可读，修改按钮置灰

### 1.9 移动端规则

- 三栏改为二层：列表页 + 详情页
- 绑定操作使用全屏抽屉
- `Bindings` 表格改成卡片列表

## 2. 设计结论

`技能中心` 是配置与治理入口，不是运行时聊天界面。
它必须让用户一眼看懂三件事：

- 系统接了什么能力
- 能力绑定给了谁
- 这些能力在项目里是否真的生效
