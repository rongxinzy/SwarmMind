# Roadmap

## Prioritization Framework

- **P0**: production readiness and trust foundations
- **P1**: decision quality and governance depth
- **P2**: ecosystem scale and adoption acceleration

## Phase Plan

| Phase | Priority | Theme | Outcomes |
|---|---|---|---|
| Phase 1 | P0 | Control-plane baseline | Goal routing, supervised execution, foundational context storage |
| Phase 2 | P0 | Governance hardening | Provenance model, permission boundary enforcement, execution trace APIs |
| Phase 3 | P1 | Policy intelligence | Adaptive routing, policy versioning, failure replay and evaluation loops |
| Phase 4 | P1 | Structured interaction | Context-to-View Compiler with typed schemas and stable UI rendering |
| Phase 5 | P2 | Enterprise connectivity | Connector framework + packaged integrations for core business systems |
| Phase 6 | P2 | Extensibility | Private skills marketplace and enterprise plugin governance |

## Immediate Milestones (Next 2 cycles)

1. Finalize context schema extensions (`provenance`, `scope`, `confidence`, `conflict_set`)
2. Deliver execution trace endpoints and replay UX baseline
3. Add approval-policy engine with risk-tiered guardrails
4. Stand up evaluation harness for routing and outcome quality

## Success Metrics

- Mean time from question to evidence-backed answer
- Approval bypass rate (lower is safer in early rollout)
- Trace completeness (% of actions linked to evidence)
- Routing policy gain vs static baseline
- Connector coverage across priority enterprise systems
